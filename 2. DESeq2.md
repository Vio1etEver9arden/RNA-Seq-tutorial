### What is DESeq2

`DESeq2` (Bioconductor) performs differential expression analysis on raw RNA-seq count data. Key features and workflow:

- Model: negative binomial modeling for count data, suitable for overdispersed RNA-seq counts.
- Normalization: estimates size factors to correct for library size differences.
- Dispersion estimation: estimates and shrinks per-gene dispersion for stability with small sample sizes.
- Differential testing: supports Wald tests and LRT; reports raw p-values and adjusted p-values (`padj`).
- Transformations: `vst()` and `rlog()` are available for visualization (PCA, heatmaps). Do NOT use transformed values for the DESeq2 statistical model — use raw integer counts for modeling.

Typical short example:

```r
# Create DESeqDataSet from raw counts
dds <- DESeqDataSetFromMatrix(
  countData = counts,
  colData = coldata,
  design = ~ condition
)

# Run DESeq2 pipeline (normalization, dispersion, model fitting)
dds <- DESeq(dds)

# Extract results for a contrast
res <- results(
  dds,
  contrast = c("condition", "treatment", "control")
)
```

Note: Input must be raw integer counts (do not input TPM/FPKM or pre-normalized values). Include relevant covariates in the design (e.g., `design = ~ batch + condition` when batch effects exist).

## Overview: workflow starting from a counts matrix

- Input: raw count matrix and a sample metadata table (`metadata`) with at least `sample` and `condition` columns.
- Counts QC: check library sizes, sample correlations, distance matrices, PCA, boxplots and density plots. Apply `vst()`/`rlog()` before visualization to stabilize variance.
- Filtering: remove genes with consistently low counts to reduce noise (e.g., keep genes with counts > X in at least N samples).
- Normalization & transformation: DESeq2 size factors for normalization; `vst()`/`rlog()` for visualization and clustering.
- Modeling & testing: specify an appropriate design in DESeq2 and run the negative binomial model to obtain p-values and `padj`.
- Multiple testing & thresholds: use BH FDR correction; common thresholds are `padj < 0.05` and `|log2FoldChange| > 1` (adjust to your experiment).
- Annotation & export: convert gene IDs (e.g., Ensembl) to gene symbols and export full and significant result tables.
- Visualization & downstream: MA-plots, volcano plots, heatmaps, sample-distance heatmaps, boxplots, GO/KEGG enrichment, PPI analysis, etc.


## DESeq2 example

### 1) Load libraries and read data
Prepare `counts.txt` (genes x samples raw counts) and `metadata.csv` (must contain `sample` and `condition`).

```r
# load the necessary libraries
library(tidyverse) # tidyverse for data manipulation
library(DESeq2)
library(clusterProfiler) # enrichment analysis
library(org.Mm.eg.db) # mouse gene annotation database
library(ggplot2) # visualization
library(ggprism) # optional plotting theme
library(ggrepel) # optional label repulsion
library(ggpubr) # optional statistical plotting helpers

# read counts table (first column is gene id)
counts <- read.table("./counts/counts.txt",
                     header = TRUE, # first row contains column names
                     row.names = 1, # first column used as row names
                     sep = "\t", # tab-separated file
                     check.names = FALSE) # do not alter column names

# read metadata (must contain sample and condition columns)
metadata <- read.csv("./metadata.csv") %>%
  mutate(condition = as.factor(condition)) # ensure condition is a factor

```

Note: `counts` rows are genes and columns are samples. `metadata` must include columns `sample` (matching counts column names) and `condition`.

### 2) Sample name cleaning and formatting (optional)
My `counts.txt` column names still include file paths and `*.sorted.bam` filenames so clean them first.

```r
counts <- counts %>%
  rename_with(
    ~ .x %>%
      basename() %>%
      str_remove("\\.sorted\\.bam$"),
    .cols = 6:last_col()
  ) # remove the directory path and the .sorted.bam suffix from the column names
```

```r
# produce short sample names like Condition_1, Condition_2 ...
metadata <- metadata %>%
  group_by(condition) %>% # group by condition
  mutate(new_sample_name = paste0(as.character(condition), "_", row_number())) %>% # generate new sample names
  ungroup()

name_map <- setNames(metadata$new_sample_name, metadata$sample) # named vector: names are original sample names, values are new names

counts <- counts %>%
  rename_with(
    ~ dplyr::recode(.x, !!!name_map),
    .cols = any_of(metadata$sample)
  ) # rename count columns based on metadata mapping
```

### 2.5) Counts QC — PCA and sample checks
Before filtering, inspect sample relationships to detect outliers or batch effects. Use `vst()` for visualization only; do not use transformed values for differential testing.

```r
# build a temporary DESeqDataSet for QC (do not run DESeq)
dds_qc <- DESeqDataSetFromMatrix(
  countData = counts[, 6:ncol(counts)], # select columns 6..end as count data
  colData = metadata %>%
    dplyr::select(new_sample_name, condition) %>%
    tibble::column_to_rownames("new_sample_name"), # set rownames from new_sample_name
  design = ~ condition # experimental design formula
)

# VST for visualization and PCA
vsd_qc <- vst(dds_qc, blind = TRUE)
pca_data <- plotPCA(
  vsd_qc,
  intgroup = "condition",
  returnData = TRUE
)
percentVar <- round(100 * attr(pca_data, "percentVar"))

pca_qc_plot <- ggplot(pca_data, aes(x = PC1, y = PC2, color = condition)) +
  geom_hline(
    yintercept = 0,
    linetype = "dashed",
    color = "black",
    size = 0.5
  ) +
  geom_vline(
    xintercept = 0,
    linetype = "dashed",
    color = "black",
    size = 0.5
  ) +
  geom_point(size = 3) +
  stat_ellipse(
    level = 0.70,
    type = "norm"
  ) +
  xlab(paste0("PC1: ", percentVar[1], "% variance")) +
  ylab(paste0("PC2: ", percentVar[2], "% variance")) +
  theme_minimal() +
  labs(title = "PCA (Counts QC)")

ggsave(
  "pca_qc_plot.png",
  plot = pca_qc_plot,
  width = 8,
  height = 6,
  dpi = 600
)
```

Note: If a sample is a clear outlier on PCA, check FastQC outputs and lab records. If the sample quality is poor, consider removing it and re-running the analysis. If batch effects drive separation, include `batch` in the design (e.g., `design = ~ batch + condition`) or consider batch-correction methods such as `ComBat` from the `sva` package (use with caution). Usually we don't remove samples unless there is a clear technical issue.

### 3) Filter low-expression genes
In this example we keep genes with counts > 10 in at least 3 samples. Adjust thresholds to your dataset.

```r
counts <- counts %>%
  filter(rowSums(across(6:last_col(), ~ .x > 10)) >= 3)
```

Note: This assumes the first 5 columns are annotation columns. If your `counts` is a pure gene x sample matrix, use `across(everything(), ...)` instead.

### 4) Build DESeq2 dataset and run differential analysis
```r
dds <- DESeqDataSetFromMatrix(
  countData = counts[, 6:ncol(counts)], # select columns 6..end as count data; consider creating a pure count matrix for clarity
  colData = metadata %>%
    dplyr::select(new_sample_name, condition) %>%
    tibble::column_to_rownames("new_sample_name"),
  design = ~ condition
)

dds <- DESeq(dds)
res <- results(dds, contrast = c("condition", "OldRecipient14", "Control14")) # adjust contrast as needed
 
results_df <- as.data.frame(res) %>%
  rownames_to_column("EnsemblID") %>%
  dplyr::mutate(regulate = case_when(
    padj < 0.05 & log2FoldChange > 1 ~ "Up",
    padj < 0.05 & log2FoldChange < -1 ~ "Down",
    TRUE ~ "Not Significant"
  ))
```


### 5) ID conversion and result export
Convert Ensembl IDs to gene symbols(names) for better interpretability. 

```r
gene_symbols <- mapIds(org.Mm.eg.db,
                       keys = results_df$EnsemblID,
                       column = "SYMBOL",
                       keytype = "ENSEMBL",
                       multiVals = "first")

results_df <- results_df %>%
  dplyr::mutate(Gene = gene_symbols) %>%
  dplyr::relocate(Gene, .before = dplyr::everything()) %>%
  dplyr::filter(!is.na(Gene))

write.csv(results_df, "deseq2_results.csv", row.names = FALSE)

# select significant genes (example thresholds: padj < 0.05 and |log2FoldChange| > 1)
sig_genes <- results_df %>% dplyr::filter(padj < 0.05 & abs(log2FoldChange) > 1)
write.csv(sig_genes, "sig_genes_all.csv", row.names = FALSE)
```

### 7) Heatmap
Plot expression heatmap for genes of interest (row-scaled). Example uses `ComplexHeatmap`.
You can select genes of interest based on significance, fold change, or known biology. Here we use a predefined list of 10 genes as an example.

```r
library(ComplexHeatmap) # differential gene heatmap
library(circlize) # color mapping
library(ggplotify) # convert ComplexHeatmap to ggplot object for saving

genes_of_interest <- c(
  "Cd209f",
  "Cd209g",
  "Awat1",
  "Cidec",
  "Cyp1a1",
  "Inmt",
  "Rag1",
  "Itln1",
  "Clca1",
  "Fer1l6"
)

selected_genes <- results_df %>%
  filter(Gene %in% genes_of_interest) %>%
  filter(EnsemblID %in% rownames(vsd))

mat_heatmap <- assay(vsd)[selected_genes$EnsemblID, ] # extract expression matrix for selected genes from vsd
rownames(mat_heatmap) <- selected_genes$Gene
mat_heatmap_scaled <- t(scale(t(mat_heatmap)))

ha <- HeatmapAnnotation(
  Condition = as.factor(colData(vsd)$condition)
) 
heatmap <- Heatmap(
  mat_heatmap_scaled,
  name = "Z-score",
  top_annotation = ha,
  cluster_rows = FALSE
)
draw(heatmap)

ggsave("heatmap.png", width = 6, height = 6)
```

### 9) Enrichment analysis (GO / KEGG)
Use `clusterProfiler`.

```r
# GO enrichment
go_res <- enrichGO(
  gene = sig_genes$Gene,
  OrgDb = org.Mm.eg.db,
  keyType = "SYMBOL",
  ont = "ALL",
  pAdjustMethod = "BH",
  pvalueCutoff = 0.05,
  qvalueCutoff = 0.05
)
write.csv(as.data.frame(go_res), "go_enrichment_results.csv", row.names = FALSE)

# visualize top 10 GO terms
go_barplot <- barplot(
    go_res,
    showCategory = 10
)

# KEGG enrichment
# KEGG enrichment requires ENTREZID, so convert gene IDs first
gene_df <- bitr(
  sig_genes$Gene,
  fromType = "SYMBOL",
  toType = "ENTREZID",
  OrgDb = org.Mm.eg.db
)
kegg_res <- enrichKEGG(
  gene = gene_df$ENTREZID,
  organism = "mmu",
  pAdjustMethod = "BH"
)
write.csv(as.data.frame(kegg_res), "kegg_enrichment_results.csv", row.names = FALSE)

# visualize top 10 KEGG pathways
kegg_barplot <- barplot(
    kegg_res,
    showCategory = 10
)
```
