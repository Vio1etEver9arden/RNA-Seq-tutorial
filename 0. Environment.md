# Environment setup

## How to Set Up a Bioinformatics Environment

The core principles of a bioinformatics environment are: **isolate by project, install tools as needed, and keep the setup reproducible**. The most reliable approach on macOS or Linux is to manage environments with `conda` or `mamba`, then install common tools through `bioconda`.

Windows is not recommended.
Users can use the Windows Subsystem for Linux (WSL) to set up a Linux environment, or they can install `conda` directly on Windows and use it to manage environments and tools. However, using WSL is generally recommended for better compatibility with bioinformatics tools.

### 1. Recommended Approach

- Create one environment per project instead of putting every tool into a single environment.
- Install the package manager first, then install bioinformatics tools.
- If something can be reproduced with an environment file, avoid installing it manually one by one.
- If you work on different directions such as RNA-seq, single-cell analysis, create separate environments.

### 2. Basic Setup

Install Miniconda or Mambaforge first, then configure the channels:

```bash
conda config --add channels defaults
conda config --add channels bioconda
conda config --add channels conda-forge
conda config --set channel_priority strict
```

If you already have `mamba`, replacing `conda` with `mamba` later will be faster.

### 3. Create a New Environment

```bash
conda create -n rnaseq_env python=3.10
conda activate rnaseq_env
```

It is a good idea to start with a stable Python version, such as 3.10 or 3.11, and then install the tools.

### 4. Common Bioinformatics Tools

If you are doing RNA-seq upstream analysis, common tools can be installed like this:

```bash
conda install -c bioconda sra-tools fastqc multiqc trim-galore hisat2 samtools subread pigz
```

If you prefer to install them step by step, you can follow this order:

```bash
conda install -c bioconda sra-tools
conda install -c bioconda fastqc multiqc
conda install -c bioconda trim-galore
conda install -c bioconda hisat2 samtools subread
```

### 5. Environment File Example

It is better to define the environment in `environment.yml` so it can be reproduced later:

```yaml
name: rnaseq_env
channels:
	- conda-forge
	- bioconda
	- defaults
dependencies:
	- python=3.10
	- sra-tools
	- fastqc
	- multiqc
	- trim-galore
	- hisat2
	- samtools
	- subread
	- pigz
```

Create the environment:

```bash
conda env create -f environment.yml
conda activate rnaseq_env
```

### 6. Export and Back Up

After the environment is ready, remember to export it:

```bash
conda env export > rnaseq_env.yml
```

This makes it easy to restore the environment when you switch computers or reinstall the system.

### 7. Verify the Setup

```bash
conda list
fastqc -h
multiqc -h
hisat2 -h
```

If these commands display help messages correctly, the environment is basically ready.

### 8. Practical Tips

- Keep data download, QC, alignment, and quantification in the same project environment when possible.
- Store reference genomes and index files in a separate directory instead of putting them inside the environment.
- For large datasets, prefer a **Linux server over a local laptop**.
- It's a little hard for beginner to set up the environment, maybe you have to spend a whole day to get it right. When you encounter issues, read the documents>> Search it in GitHub>> ask ChatGPT.
